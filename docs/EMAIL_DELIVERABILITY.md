# 이메일 발송 설정 및 스팸 분류 검토

Sales Factory의 `send_emails.py`로 발송하는 B2B 제안 이메일이 **스팸함으로 분류되지 않도록** 꼼꼼히 점검한 내용입니다.

---

## 1. 현재 발송 구조 요약

| 항목 | 현재 설정 |
|------|-----------|
| SMTP | Gmail (smtp.gmail.com:587, STARTTLS) |
| 발신자 | `SMTP_USER` (예: yeomjw0907@onecation.co.kr) → From: `Onecation <...>` |
| 본문 | HTML + Plain text 멀티파트, 한국어, 회사 맞춤 제안 문구 |
| 첨부 | D1 터치 시에만 PDF 1개 (제안서) |
| 발송 간격 | 연속 발송 시 약 3초 대기 (Gmail 제한 완화 목적) |

---

## 2. 스팸으로 갈 수 있는 요인 (위험도별)

### 🔴 높은 위험

- **발신 도메인 인증 부재**  
  `@onecation.co.kr` 등 **자사 도메인**으로 보내는데 **SPF/DKIM/DMARC**가 없으면 대형 메일함(Gmail, Naver, Daum 등)이 “의심 스팸”으로 쉽게 분류합니다.  
  → **반드시** SPF·DKIM 설정 후, 여유 있으면 DMARC까지 추가하는 것을 권장합니다.

- **Gmail 개인 계정으로 대량 발송**  
  Gmail은 “개인 사용” 가정이라 하루 발송량·패턴이 다르면 스팸/차단 위험이 있습니다.  
  → 가능하면 **Google Workspace(구 G Suite)** 또는 **전용 SMTP 서비스**(SendGrid, Amazon SES, Mailgun 등) 사용이 안전합니다.

- **수신자 리스트 품질**  
  이메일 주소 오타, 비존재 도메인, 스팸트랩 주소가 섞여 있으면 반송·스팸 클릭이 늘어나고, 도메인/IP 평판이 떨어져 이후 메일이 스팸 처리되기 쉽습니다.  
  → `contacts.csv`는 실제 담당자·사용 가능 주소만 넣고, 가능하면 한 번 검증(존재 여부·역할)하는 것이 좋습니다.

### 🟡 중간 위험

- **제목/본문에 스팸 유발 표현**  
  “무료”, “당첨”, “즉시”, “100%”, 과도한 대문자·특수문자, URL만 나열하는 식의 문구는 필터에 걸리기 쉽습니다.  
  → 현재는 **맞춤 제안·다음 단계 안내** 위주라 상대적으로 양호하지만, 에이전트가 생성하는 문구는 가끔 점검하는 것이 좋습니다.

- **첨부 파일**  
  PDF 1개만 첨부하고, 파일명이 한글/특수문자일 때 일부 필터가 민감하게 반응할 수 있습니다.  
  → 코드에서 이미 `filename=...encode("utf-8").decode("ascii", errors="replace")`로 정리하고 있어 위험은 낮은 편입니다. 실행 시 로그에서 파일명이 깨지지 않게만 확인하면 됩니다.

- **발송 속도**  
  짧은 시간에 너무 많이 보내면 “봇/대량 발송”으로 보일 수 있습니다.  
  → 현재 3초 대기는 기본적인 완화일 뿐, **하루 50~100통 이상**이면 구간을 나누어 보내거나(예: 10분에 20통) 전용 SMTP로 옮기는 것을 고려하세요.

### 🟢 현재 구조에서 상대적으로 양호한 점

- **HTML + Plain text** 멀티파트로 보내고 있어 텍스트 비율·구조가 자연스러움.
- **수신 거부 안내** 문구가 푸터에 있어 법적·정책적 요구에 부합.
- **회사명·담당자명** 치환으로 개인화되어 있어 “일괄 스팸” 패턴과는 다름.
- **고정된 발신자(Onecation)** 로 비즈니스 제안 목적이 분명함.

## 3. 스팸 방지를 위한 권장 조치 (체크리스트)

### 3.1 도메인 인증 (필수에 가깝게 권장)

| 항목 | 설명 |
|------|------|
| **SPF** | 발송하는 SMTP 서버 IP/호스트만 “허용”하도록 DNS TXT에 기록. (Gmail 사용 시 Google SPF 포함) |
| **DKIM** | 메일에 서명을 붙여 “이 도메인이 보낸 메일이 맞다”고 증명. 대형 메일함이 신뢰도 판단에 많이 사용. |
| **DMARC** | SPF/DKIM 실패 시 어떻게 할지(거부/격리 등) 정책을 DNS에 명시. 도메인 악용 방지 + 평판 관리에 유리. |

- **Gmail/Workspace** 사용 시: Google 도메인 설정 가이드에서 SPF/DKIM 설정 방법 확인.  
- **onecation.co.kr**을 발신 도메인으로 쓸 경우: 해당 도메인 DNS 관리처에서 SPF/DKIM(DMARC) 레코드 추가.

### 3.2 발송 환경

- [ ] **Gmail 개인**이면 가능한 한 **Google Workspace**로 이전하거나,  
  **SendGrid / Amazon SES / Mailgun** 등 전용 SMTP로 발송 계정 분리.
- [ ] `SMTP_USER`는 **실제 사용하는 비즈니스 이메일**로 통일 (예: `contact@onecation.co.kr`).
- [ ] **앱 비밀번호** 사용 시 2단계 인증 켜져 있는지 확인.

### 3.3 수신자 및 발송 패턴

- [ ] **contacts.csv**는 오타·비존재 주소 제거, 가능하면 이메일 검증 툴로 1회 검증.
- [ ] 하루 발송량이 많으면 **시간대 분산** (예: 오전/오후 나누기, 10~20통 단위로 간격 두기).
- [ ] **반송·스팸 신고**가 나오는 주소는 리스트에서 제거하고, 같은 주소로 반복 발송하지 않기.

### 3.4 본문·제목

- [ ] 제목에 **회사명·제안 목적**이 드러나도록 유지 (현재 `[회사명] 맞춤 마케팅 제안 — D1` 형태는 적절함).
- [ ] 에이전트가 생성하는 **outreach_emails.md** 문구에 “무료/당첨/즉시/100%” 등 과장 표현이 들어가지 않도록 태스크 지시나 사후 검토 추가.
- [ ] 본문에 **의미 있는 텍스트 비율** 유지 (이미지/링크만 많은 메일은 피하기).

### 3.5 기술적 점검

- [ ] **Reply-To**를 발신자와 동일하거나 수신자가 답장하기 쉬운 주소로 설정 (필요 시 `send_emails.py`에서 `Reply-To` 헤더 추가).
- [ ] **List-Unsubscribe** 헤더는 대량 뉴스레터에 가깝게 쓸 때 유리할 수 있음. B2B 1:1 제안에는 필수는 아님.
- [ ] PDF 첨부는 **D1만** 유지 (현재처럼)하고, 파일 크기가 지나치게 크지 않게 유지.

## 4. 요약: 스팸 분류될 가능성과 대응

| 상황 | 스팸 위험 | 권장 대응 |
|------|-----------|-----------|
| SPF/DKIM 없이 자사 도메인(@onecation.co.kr) 발송 | **높음** | SPF·DKIM 반드시 설정 |
| Gmail 개인 계정으로 일 50통 이상 발송 | **중~높음** | Workspace 또는 전용 SMTP로 이전, 발송량·간격 조절 |
| 수신자 리스트에 오타·비존재·스팸트랩 포함 | **중간** | 리스트 정제·검증, 반송/스팸 주소 제거 |
| 제목/본문에 과장·스팸 유발 단어 다수 | **중간** | 문구 가이드라인 정리, 생성 문구 검토 |
| 현재처럼 맞춤 제안 + HTML/plain + 푸터 안내 | **상대적으로 낮음** | 유지하되, 위 항목들 보완 시 더 안정적 |

**정리:**  
- **도메인 인증(SPF/DKIM, 가능하면 DMARC)** 과 **적절한 발송량·수신자 품질**이 가장 중요합니다.  
- 현재 코드 구조(발신자 표시, 본문 구성, 첨부 방식)는 B2B 제안 메일로서 무난한 편이므로, **인프라(도메인·SMTP)·운영(리스트·발송 패턴)** 쪽을 보강하면 스팸 분류 가능성을 꽤 줄일 수 있습니다.
